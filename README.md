A minimal self-hosted URL shortener written in Go. Public users follow short links and get a redirect; the admin manages links via a web panel.

## Stack

- **Go** — single binary
- **SQLite** — database (`u.db`), no external dependencies
- **chi** — HTTP router
- Templates and static assets embedded into the binary via `go:embed`

## Development

```bash
make gen-config             # generate config.yaml with a random cookie_key
make run-dev PASSWORD=xxx   # build and run (default PASSWORD=dev)
make run PASSWORD=xxx       # build and run (PASSWORD required)
make test                   # run all tests
make build                  # build binary only
make hashpw PASSWORD=xxx    # generate bcrypt hash for use in config
```

## Configuration

### Generate config

```bash
make gen-config SITE_URL=https://u.example.com ADMIN_USER=admin
```

This creates `config.yaml` with a random `cookie_key`. The admin password is **not** stored in the file — pass it at runtime via `U_ADMIN_PASSWORD`.

### Config file reference

| Field | Default | Description |
|---|---|---|
| `site_url` | `http://localhost:8080` | Public base URL |
| `addr` | `:8080` | Address to listen on |
| `db_path` | `./u.db` | Path to SQLite database file |
| `cookie_key` | *(required)* | Secret for HMAC session cookies — auto-generated by `make gen-config` |
| `admin.username` | *(required)* | Admin login |
| `admin.password` | *(optional)* | Pre-computed bcrypt hash — use env var instead |

### Admin password

Pass the password via the `U_ADMIN_PASSWORD` environment variable. The app accepts either plaintext (hashed at startup) or a pre-computed bcrypt string:

```bash
# Plaintext — hashed automatically at startup
U_ADMIN_PASSWORD=yourpassword ./u

# Pre-hash with make (for docker-compose, scripts, etc.)
make hashpw PASSWORD=yourpassword
# → $2a$10$...  (paste into docker-compose env)
```

## Production Deployment

### Directory layout

```
~/
├── docker-compose-caddy/       # Caddy (separate project, already running)
│   ├── Caddyfile
│   └── docker-compose.yml
└── docker-compose-u/           # URL Shortener
    ├── docker-compose.yml
    ├── docker-compose.override.yml  # local overrides (not tracked by git)
    ├── cron/                        # backup container (copy from repo)
    └── data/
        ├── config.yaml
        └── u.db
```

### 1. Create the shared Docker network (once)

```bash
docker network create common
```

### 2. Configure Caddy

Add a site block to `Caddyfile`:

```caddyfile
https://u.example.com:443 {
    encode gzip

    handle {
        reverse_proxy u:8080 {
            header_up Host {host}
            header_up X-Real-IP {http.request.remote_host}
            header_up X-Forwarded-For {http.request.remote_host}
            header_up X-Forwarded-Proto {scheme}
            header_up X-Forwarded-Ssl on
        }
    }

    header {
        X-Content-Type-Options nosniff
        X-Frame-Options DENY
        X-XSS-Protection "1; mode=block"
        Referrer-Policy strict-origin-when-cross-origin
    }
}
```

Reload Caddy:

```bash
cd ~/docker-compose-caddy
docker compose exec caddy caddy reload --config /etc/caddy/Caddyfile
```

### 3. Set up the app directory

```bash
mkdir -p ~/docker-compose-u/data
cd ~/docker-compose-u
touch data/u.db
```

Copy the `cron` directory from this repo:

```bash
cp -r /path/to/u/cron ~/docker-compose-u/cron
```

Or clone just that directory via sparse checkout:

```bash
git clone --filter=blob:none --sparse https://github.com/rhamdeew/u.git _tmp
git -C _tmp sparse-checkout set cron
cp -r _tmp/cron ~/docker-compose-u/cron
rm -rf _tmp
```

Create `data/config.yaml`:

```yaml
site_url: "https://u.example.com"
listen: ":8080"
db_path: "/data/u.db"
cookie_key: "replace-with-a-long-random-secret"
admin:
  username: "admin"
  password: "$2a$12$..."   # bcrypt hash from: ./u hashpw yourpassword
```

Create `docker-compose.yml`:

```yaml
services:
  app:
    image: ghcr.io/rhamdeew/u:latest
    restart: unless-stopped
    networks:
      common:
        aliases:
          - u
    volumes:
      - ./data:/data
    environment:
      U_DB_PATH: /data/u.db

  cron:
    build:
      context: ./cron
      dockerfile: Dockerfile
    container_name: u_cron
    volumes:
      - ./data:/data:ro
      - backup_data:/backups
    environment:
      S3_BUCKET: ""
      S3_ACCESS_KEY: ""
      S3_SECRET_KEY: ""
      S3_HOST: ""
      S3_REGION: ""
    restart: unless-stopped

networks:
  common:
    external: true

volumes:
  backup_data:
```

Start:

```bash
docker compose up -d
```

## Backups

The `cron` container backs up `u.db` daily at **07:00 UTC** and stores archives in the `backup_data` Docker volume.

Rotation policy: **1 monthly**, **2 weekly** (Mondays), **3 daily**.

To enable S3 sync, set credentials via `docker-compose.override.yml` (not tracked by git):

```yaml
# docker-compose.override.yml
services:
  cron:
    environment:
      S3_BUCKET: "my-bucket"
      S3_ACCESS_KEY: "AKIAXXXXXXXX"
      S3_SECRET_KEY: "secret"
      S3_HOST: "s3.amazonaws.com"
      S3_REGION: "us-east-1"
```

View backup logs:

```bash
docker compose logs cron
```

Run a backup manually:

```bash
docker compose exec cron /usr/local/bin/backup.sh
```
