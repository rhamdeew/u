# CLAUDE.md — Developer Reference for `u`

`u` is a self-hosted URL shortener written in Go. It is a reimplementation of
[YOURLS](https://yourls.org/) without PHP or MySQL: a single binary, SQLite
database, server-side rendered HTML templates.

---

## Quick start

```bash
make gen-config                   # writes data/config.yaml with a random cookie_key
make run-dev PASSWORD=secret      # build + run (dev shortcut; PASSWORD defaults to "dev")
make run     PASSWORD=secret      # build + run (PASSWORD required)
make test                         # run all tests
make build                        # build binary only (outputs ./u)
make hashpw  PASSWORD=secret      # print bcrypt hash (for config.yaml or docker-compose)
```

The server listens on `:8080` by default. Open `http://localhost:8080/admin`.

## Database schema

Managed by `db.migrate()` in `internal/db/db.go` via `CREATE TABLE IF NOT EXISTS`.
No external migration tool is used.

```sql
CREATE TABLE links (
    keyword    TEXT PRIMARY KEY,
    url        TEXT NOT NULL,
    title      TEXT NOT NULL DEFAULT '',
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    ip         TEXT NOT NULL DEFAULT '',
    clicks     INTEGER NOT NULL DEFAULT 0
);

CREATE TABLE clicks (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    keyword    TEXT NOT NULL REFERENCES links(keyword) ON DELETE CASCADE,
    clicked_at TEXT NOT NULL DEFAULT (datetime('now')),
    referrer   TEXT NOT NULL DEFAULT '',
    user_agent TEXT NOT NULL DEFAULT '',
    ip         TEXT NOT NULL DEFAULT ''
);

-- Auto-incrementing counter for keyword generation
CREATE TABLE counter (
    id       INTEGER PRIMARY KEY CHECK (id = 1),
    next_val INTEGER NOT NULL DEFAULT 1
);
```

- `links.clicks` is a **denormalised counter** kept in sync by `IncrClicks()`.
- `clicks` stores raw per-hit data used only for the stats detail panel.
- Timestamps are stored as `TEXT` in `"2006-01-02 15:04:05"` UTC format.
- SQLite is opened in WAL mode with `SetMaxOpenConns(1)` (single writer).

## Configuration

Loaded from a YAML file (default `data/config.yaml`). Every field can be
overridden with an environment variable.

| YAML key            | Env var               | Default                  | Notes |
|---------------------|-----------------------|--------------------------|-------|
| `site_url`          | `U_SITE_URL`          | `http://localhost:8080`  | Trailing slash stripped |
| `db_path`           | `U_DB_PATH`           | `./u.db`                 | |
| `cookie_key`        | `U_COOKIE_KEY`        | *(required)*             | 64-char hex generated by `make gen-config` |
| `addr`              | `U_ADDR`              | `:8080`                  | |
| `admin.username`    | `U_ADMIN_USERNAME`    | *(required)*             | |
| `admin.password`    | `U_ADMIN_PASSWORD`    | *(required)*             | Accepts plaintext (hashed at startup) or `$2a$…` bcrypt hash |

The password is **never stored in the config file** by convention — use
`U_ADMIN_PASSWORD` at runtime.

## HTTP routes

### Public

| Method | Path          | Handler         | Description |
|--------|---------------|-----------------|-------------|
| GET    | `/{keyword}`  | `redirect.Handler` | 301 redirect; logs click asynchronously; 404 if unknown |
| GET    | `/`           | `redirect.Handler` | Redirects to `/admin` (logged in) or `/admin/login` |
| GET    | `/static/*`   | `http.FileServer` | Embedded CSS/assets |

### Admin (all except login require session cookie)

| Method | Path                                       | Handler               |
|--------|--------------------------------------------|----------------------|
| GET    | `/admin/login`                             | `getLogin`            |
| POST   | `/admin/login`                             | `postLogin`           |
| POST   | `/admin/logout`                            | `postLogout`          |
| GET    | `/admin`                                   | `getIndex`            |
| POST   | `/admin/links`                             | `postCreateLink`      |
| GET    | `/admin/links/{keyword}/edit`              | `getEditLink`         |
| POST   | `/admin/links/{keyword}/edit`              | `postEditLink`        |
| POST   | `/admin/links/{keyword}/delete`            | `postDeleteLink`      |
| GET    | `/admin/links/{keyword}/stats`             | `getStats`            |
| GET    | `/admin/links/{keyword}/stats/{date}/details` | `getClickDetails` (JSON) |

`{date}` is `YYYY-MM-DD`. The `details` endpoint returns JSON used by the
stats page's expandable row UI (fetched client-side on demand).

## Authentication

Session cookies are **stateless** (no server-side store):

```
cookie value = "<username>|<unix_expiry>|<hmac-sha256-hex>"
```

- `auth.SetSession` — writes signed cookie (24h TTL).
- `auth.GetSession` — verifies HMAC + expiry, returns username or `""`.
- `auth.CheckPassword` — `bcrypt.CompareHashAndPassword`.
- `auth.HashPassword` — used by `hashpw` subcommand and startup env-var flow.

## Keyword generation

Keywords are **base-36 encoded integers** (`[0-9a-z]`). The counter lives in
the `counter` table:

1. `INSERT` into `links` inside a transaction.
2. `SELECT next_val FROM counter WHERE id = 1`.
3. `UPDATE counter SET next_val = next_val + 1`.
4. `db.EncodeBase36(n)` converts the integer to the keyword string.

Custom keywords bypass this and are inserted directly.

## Templates

All templates are embedded into the binary via `//go:embed` in `web/web.go`.
They use Go's `html/template` package with a shared `FuncMap`:

| Function      | Description |
|---------------|-------------|
| `formatTime`  | `time.Time` → `"2006-01-02 15:04"` or `"—"` for zero value |
| `truncate`    | Truncate a string to N runes, appending `…` |
| `add`         | Integer addition (for pagination) |
| `sub`         | Integer subtraction |
| `seq`         | Returns `[]int{1…n}` |

Each page is a separate `*template.Template` parsed from `base.html` + the
page file. Templates define blocks: `title`, `header-right`, `content`.

## Stats detail panel

The stats page (`/admin/links/{keyword}/stats`) shows clicks grouped by day.
Clicking a date row expands an inline panel that lazily fetches individual
click records (IP, user-agent, referrer, timestamp) via:

```
GET /admin/links/{keyword}/stats/{date}/details
→ JSON: [{"clicked_at":"…","ip":"…","user_agent":"…","referrer":"…"}, …]
```

The panel is built with vanilla JS (no framework). Data is fetched once per
date and cached in `dataset.loaded` to avoid repeat requests.

## Click recording

Clicks are recorded **asynchronously** in `redirect.Handler.ServeHTTP` to
avoid adding latency to the redirect:

```go
go func() {
    _ = h.db.InsertClick(db.ClickRecord{…})
    _ = h.db.IncrClicks(keyword)
}()
```

Both `InsertClick` and `IncrClicks` are safe under SQLite's single-writer
model because the goroutine serialises through the `*sql.DB` connection pool
(which has `MaxOpenConns = 1`).

## Adding a new admin page

1. Create `internal/web/templates/mypage.html` with `{{define "title"}}`, `{{define "content"}}`.
2. Add `MyPage *template.Template` to `admin.Templates` in `admin.go`.
3. Parse it in `web.ParseTemplates()` with `parse("templates/mypage.html")`.
4. Add a handler method on `*admin.Handler` and register the route in `Routes()`.

## Docker / deployment

```bash
# Build image
docker build -t u .

# Production via docker-compose (see README.md for full setup)
docker compose up -d
```

The `Dockerfile` is a two-stage build:
1. `golang:1.25-alpine` — compiles the binary.
2. `alpine:latest` — minimal runtime image with the binary + no source.

The SQLite database (`u.db`) and `config.yaml` are mounted from the host via
a volume at `/data`.

The `cron/` container backs up `u.db` daily using `rclone` to S3-compatible
storage with rotation: 1 monthly, 2 weekly, 3 daily.
